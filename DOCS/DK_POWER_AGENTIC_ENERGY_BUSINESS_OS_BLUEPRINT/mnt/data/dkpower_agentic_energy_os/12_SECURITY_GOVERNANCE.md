# Security & Governance
# DK Power Agentic Energy Business OS

## 1. Security Principles

- Least privilege access
- Human approval for AI financial output
- Audit trail for critical action
- Encryption for customer data
- No hidden decision by AI

## 2. Roles

| Role | Access |
|---|---|
| Owner | All modules |
| Sales | Leads, opportunities, proposals |
| Engineer | Technical review, proposal detail |
| Project Manager | Projects, tasks |
| Admin Finance | Quotations, invoices |
| Customer | Own portal only |

## 3. Critical Controls

- MFA for internal users
- Role Based Access Control
- Audit logs
- Approval workflow
- PDF versioning
- AI output disclaimer
- Manual override log

## 4. AI Governance

AI cannot:

- Approve final price
- Sign contract
- Promise guaranteed savings
- Change margin rules
- Delete customer records

AI can:

- Draft proposal
- Summarize lead
- Suggest system sizing
- Flag risk
- Prepare project tasks
