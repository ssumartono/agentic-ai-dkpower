# Developer Handoff
# DK Power Agentic Energy Business OS

## 1. Build Priority

Start with the money-flow spine:

Lead → Opportunity → Solar Calculation → Proposal → Review → Quotation → Project

## 2. Suggested Folder Structure

```text
src/
  app/
  components/
  modules/
    leads/
    customers/
    opportunities/
    proposals/
    projects/
    dashboard/
    agents/
  lib/
  services/
  database/
  types/
```

## 3. First Sprint Tasks

- Create repo
- Configure auth
- Create database schema
- Build layout
- Build lead list
- Build lead detail
- Build customer table
- Build opportunity table

## 4. Second Sprint Tasks

- Solar calculator
- Product catalog
- Price list
- Proposal data model
- AI lead summarizer
- Proposal draft screen

## 5. Third Sprint Tasks

- PDF generator
- Technical review
- Dashboard
- Project creation
- UAT fixes

## 6. Definition of Done

A feature is done when:

- UI works
- API works
- Database persists
- Error state handled
- Role access checked
- Audit log created for critical action
- User can test without developer help
